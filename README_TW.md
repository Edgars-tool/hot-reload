# Hot-Reload 外掛 for Obsidian.md 外掛

如果您開發 [Obsidian.md](https://Obsidian.md/) 的外掛，您可能熟悉需要頻繁重啟/重新載入 Obsidian，或者進入其外掛設定來停用並重新啟用您正在開發的外掛。

現在，您不再需要這麼做了。只要安裝這個外掛，讓它完成所有工作。啟用後，此外掛會自動監控任何包含 `.git` 子目錄或名為 `.hotreload` 之檔案的外掛目錄中的 `main.js` 或 `styles.css` 變更，然後在變更停止約四分之三秒後自動停用並重新啟用該外掛。重新載入時，一個通知會短暫出現在 Obsidian 中。（開發者工具中的詳細日誌也會反映這一點。）

此外掛還會監控 `manifest.json` 檔案的變更，以及 `.git` 或 `.hotreload` 的新增或移除，並據此更新其監控清單。因此，向熱重載清單新增或移除外掛不需要任何設定：只需將您的新外掛置於版本控制之下，或在其中建立 `.hotreload` 檔案。

（而且，由於 Obsidian 只會在 外掛下載中包含 `main.js` 和 `styles.css`，您不必擔心 `.hotreload` 會洩漏到外部：當其他人從 Obsidian 市場下載您的外掛時，即使他們也使用了熱重載外掛，也不會被熱重載。）

不過，請注意，如果您啟用了此外掛，它 *可以* 啟用目前在設定中未啟用的外掛，只要您已使用 git 檢出它們或已新增 `.hotreload` 檔案。（對於開發來說，這實際上是好事，因為當您進行的變更破壞了外掛載入程序時，您只需要再次儲存您的檔案，熱重載就會嘗試重新啟用它，讓您不必重新載入或再次回到設定。）

最後，請注意，雖然此外掛負責處理重載您外掛的繁重工作，但請記住，*您外掛* 的責任是正確地清理自己。如果您沒有很好地利用 `onunload()` 和各種 `registerX()` 方法來確保所有您的變更正確卸載，您可能會讓 Obsidian 處於不穩定狀態，迫使您重啟或重新載入才能將應用程式恢复到可用狀態。

### 安裝

這是像任何其他外掛一樣的 Obsidian 外掛，必須將其克隆或解壓縮到您的 vault 的 `.obsidian/plugins/` 目錄（或者如果您在 Obsidian 設定中覆蓋了 `.obsidian` 目錄名稱，則為類似的目錄），然後在 Obsidian 設定中啟用。它未註冊為標準社群外掛以便在 Obsidian 內部下載或更新，因為它僅供開發者使用，並且因為它可以啟用其他外掛。

### 符號連結

如果您希望有一個外掛資料夾連結到您的建置，熱重載支援目錄符號連結。然而，通常以另一種方式執行會更簡單，例如 `ln -s /some/Vault/.obsidian/plugins/my-plugin ./dist` 讓您的建置器寫入您的開發/測試 vault。不支援檔案符號連結。

### 行動裝置外掛開發

此外掛專為桌面版設計，其行動裝置支援（於 0.3.0 新增）仍處於實驗/測試階段，並且僅支援 Obsidian 1.6.7 及以上版本。根據您使用的同步工具/服務，您可能還想嘗試：

- Obsidian Sync 服務：啟用 Obsidian 的「同步 > Vault 設定同步、已安裝的社群外掛」設定，以在同步後自動重新啟動修改過的外掛（感謝 @ChasKane 提供此技巧）。（注意：如果您這樣做，您應該 *不要* 在行動裝置上啟用熱重載，否則您將會有雙重重新載入修改過的外掛的問題！）

- 其他同步工具：[Hot Reload Mobile](https://github.com/shabegom/obsidian-hot-reload-mobile) 外掛可與任何更新筆記的同步機制配合使用，方法是寫入桌面上的筆記以在筆記同步後觸發行動裝置上的重新載入。
